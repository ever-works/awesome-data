## Overview

awesome-inspiration is a curated, open-source archive of design and development inspiration. It aggregates links and resources useful for designers and developers, focusing on real-world examples of interfaces, flows, and patterns across the web.

## Features

- **Curated inspiration archive**
  - Focused on high-quality examples rather than exhaustive listings
  - Targeted at both designers and developers
  - Aimed at helping elevate visual design, UX, and product quality

- **Organized by themes and patterns** (via Table of Contents)
  - **Landing** – landing page layouts and hero sections
  - **UI & UX** – user interface patterns and user experience flows
  - **SaaS** – SaaS product sites, pricing layouts, dashboards, etc.
  - **Onboarding** – signup, activation, and onboarding flows
  - **Component** – individual UI components and modules
  - **Web** – general website inspiration
  - **eCommerce** – product pages, carts, and shop experiences
  - **Mobile** – mobile UI and responsive patterns
  - **Dark** – dark-mode / dark-themed designs
  - **Navbar** – navigation bars and header patterns
  - **Footer** – footer layouts and content structures
  - **404** – error / not-found page designs
  - **Deck** – presentation decks and slide design
  - **OG** – Open Graph / social share card examples
  - **Arts** – artful, experimental, or visually expressive works
  - **Logo** – logo design inspiration
  - **Design** – general design references and resources
  - **Development** – development-related resources and references
  - **Interface** – interface layouts and interaction patterns
  - **Portfolio** – personal and agency portfolio sites
  - **Archive** – broader archives and collections of inspiration
  - **Web3** – Web3 / crypto / blockchain related UI examples

- **Web-based directory**
  - Public site generated from the repository (`index.html`)
  - Simple, browsable interface for exploring categories

- **Open-source repository structure**
  - `README.md` – main description and categorized links
  - `index.html` – website front-end for the archive
  - `code-of-conduct.md` – expected behavior for contributors
  - `contributing.md` – guidelines on how to suggest or add resources

- **Community-driven curation**
  - Contributions are explicitly welcomed
  - Clear instructions via `contributing.md` for adding new links and categories

## Use Cases

- Discover layouts and UI patterns before starting a new project
- Gather references for landing pages, SaaS products, or mobile apps
- Explore real-world onboarding, navigation, and 404 implementations
- Build moodboards or design direction documents
