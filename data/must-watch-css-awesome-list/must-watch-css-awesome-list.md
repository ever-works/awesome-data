# Must-Watch CSS (Awesome List)

A curated, open-source collection of notable CSS talks and presentations for front-end developers and designers.

**Website:** https://github.com/AllThingsSmitty/must-watch-css#readme

---

## Overview

Must-Watch CSS is an "awesome list"–style GitHub repository that aggregates well-received conference talks and presentations focused on CSS. It spans modern and foundational topics, helping developers and designers deepen their understanding of CSS techniques, performance, and tooling.

---

## Features

- **Curated CSS Talk Collection**  
  - Hand-picked, well-received talks specifically about CSS.  
  - Focuses on quality and relevance for front-end developers and designers.

- **Wide Range of CSS Topics**  
  Talks cover areas including:
  - CSS Grid
  - Flexbox
  - Custom properties (CSS variables)
  - CSS performance
  - CSS frameworks
  - Sass
  - Tooling / build tools and related workflows
  - General and advanced CSS techniques

- **Organized by Year**  
  Talks are grouped chronologically, making it easy to explore content by era or keep up with newer material:
  - 2025
  - 2024
  - 2022
  - 2020
  - 2019
  - 2018
  - 2017
  - 2016
  - 2015
  - 2014
  - 2013
  - 2012
  - 2010

- **Runtime Indicators**  
  - Each entry lists video duration (e.g., `30:03`, `55:55`, `59:06`), helping you gauge time commitment at a glance.

- **Open Source Repository**  
  - Hosted on GitHub (`AllThingsSmitty/must-watch-css`).
  - Includes:
    - `README.md` with the main list and structure.
    - `LICENSE` file specifying usage terms.
    - `CODE-OF-CONDUCT.md` to guide community behavior.
    - `CONTRIBUTING.md` with guidelines for submitting new talks or improvements.
    - `assets/img` with repository imagery (e.g., TV icon graphic).

- **Community & Ecosystem Links**  
  - References a sibling list: **Must-Watch JavaScript** (for JavaScript talks).  
  - Connects to the broader **Awesome Lists** ecosystem via sindresorhus’s curated lists.

- **Contribution-Friendly**  
  - Clear contribution instructions and standards via `CONTRIBUTING.md`.  
  - Code of conduct encourages inclusive, respectful participation.

- **Licensing & Governance**  
  - Explicit license for reuse and sharing (details in `LICENSE`).  
  - Defined code of conduct for contributors and maintainers.
