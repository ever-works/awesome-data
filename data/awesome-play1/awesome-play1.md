## Features

### 1. Modules Directory

A categorized list of Play 1.x modules, each annotated with badges that describe its status and distribution.

**Module badge system:**
- **Registered on playframework.com/modules (registered):**  
  Indicates that a module is officially registered in the legacy Play module registry. The badge links to its page on `playframework.com/modules`.
- **Not registered on playframework.com/modules (external repo):**  
  Indicates that the module is not in the legacy registry and must be added via an external repository in `dependencies.yml`. The badge links to the official module page (typically a GitHub repo).
- **Mavenized:**  
  Shows that the module is available on Maven Central, typically via the `maven-play-plugin`. The badge links directly to the Maven repository artifact page.
- **Updated since registry freeze:**  
  Marks modules that have been updated after the official Play 1.x module registry was frozen, with the badge linking to the current official module page.

These badges help developers quickly understand:
- Where to fetch the module from
- How it is distributed (legacy registry vs external vs Maven Central)
- Whether it is actively maintained beyond the frozen registry
