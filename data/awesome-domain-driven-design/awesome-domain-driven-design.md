# Awesome Domain-Driven Design

## Overview
Awesome Domain-Driven Design is a curated directory of resources focused on:

- Domain-Driven Design (DDD)
- Command Query Responsibility Segregation (CQRS)
- Event Sourcing
- Event Storming

It aggregates educational and practical materials such as books, articles, patterns, and example implementations for building complex software systems with rich domain models.

## Features
- **Curated resource list**
  - Hand-picked links specifically focused on DDD and closely related architectural patterns.
  - Emphasis on quality and relevance rather than volume.

- **Domain-Driven Design coverage**
  - Introduces DDD as an approach to handling complex business domains.
  - Focus on:
    - Core domain and domain logic
    - Model-driven design for complex systems
    - Continuous refinement of a domain model
    - Collaboration between technical and domain experts

- **Related architectural patterns**
  - Resources for **CQRS (Command Query Responsibility Segregation)**.
  - Resources for **Event Sourcing**.
  - Resources for **Event Storming**.

- **Learning-oriented materials**
  - Includes books and long-form reading.
  - Articles and blog posts for targeted topics.
  - Patterns and practices for structuring domain models and boundaries.
  - Example implementations showing DDD and related patterns in real codebases.

- **Community and contribution structure**
  - Code of conduct file for community interaction guidelines.
  - Contributing guidelines for proposing new entries or improvements.
