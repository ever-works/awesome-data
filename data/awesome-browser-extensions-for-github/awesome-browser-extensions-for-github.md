## Overview

Awesome Browser Extensions for GitHub is a curated, open-source directory of browser extensions that enhance the GitHub experience. It focuses on UI improvements, productivity tools, and integrations for developers using GitHub in the browser.

## Features

- **Curated extension list**
  - Collection of browser extensions specifically designed for GitHub.
  - Focus on UI enhancements, workflow improvements, and integrations.

- **Multi-browser coverage**
  - Usage statistics aggregate extension usage across all supported browsers.

- **Daily-updated metrics**
  - Tracks and updates, on a daily basis:
    - Overall usage of each extension (across browsers).
    - Last publish date of each extension.
    - GitHub stars of the extensionâ€™s repository (where applicable).

- **Open-source repository**
  - Hosted on GitHub with public files such as `data.json`, `generate-readme.js`, and others that power the list generation and automation.

- **Automated README generation**
  - Uses scripts (e.g., `generate-readme.js`, `update.js`) to automatically build and refresh the main README from structured data.

- **Structured extension data**
  - Central `data.json` file stores metadata about all listed extensions, enabling consistent formatting and automated updates.

- **Contribution workflow**
  - Codeless contribution: new extensions can be submitted simply by opening a GitHub Issue.
  - Additional contribution guidelines available in `contributing.md`.

- **Awesome list compliant**
  - Part of the broader "Awesome" ecosystem (referenced via sindresorhus/awesome), following common awesome-list conventions for structure and quality.

## Pricing

- **Free**: The directory and all its content are available to browse and use at no cost.
