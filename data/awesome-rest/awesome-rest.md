## Description
Awesome REST is a collaboratively maintained, curated list of resources focused on RESTful API architecture, development, testing, and performance. It aggregates links to tools, libraries, frameworks, tutorials, and reference material for building and consuming REST APIs.

## Features
- **Curated resource collection**
  - Focused specifically on RESTful API concepts and practice
  - Community-maintained and open to contributions
  - Organized by topic and technology for easier navigation

- **Design resources**
  - General REST API design references
  - **Guidelines** section with best practices, patterns, and conventions for designing REST APIs

- **Standards**
  - Links to specifications and standards relevant to RESTful APIs
  - Includes protocols, formats, and common conventions used in REST-based systems

- **Clients section** (tools and libraries for consuming REST APIs)
  - **PHP clients**: libraries and tools for calling REST APIs from PHP
  - **Client-side JavaScript clients**: browser-focused REST clients and helpers
  - **Node.js clients**: REST client libraries for server-side JavaScript
  - **Ruby clients**: gems and tools for interacting with REST services
  - **Go clients**: Go libraries for consuming REST endpoints
  - **.NET clients**: .NET and C# libraries for REST communication
  - **Client generators**: tools that generate client code from API descriptions (e.g., OpenAPI/Swagger-style generators and similar tools)

- **Servers section** (frameworks and tools for building REST APIs)
  - **Directly on top of an RDBMS**: tools that expose relational database data directly as REST endpoints
  - **Node.js servers**: frameworks and libraries for building REST APIs in Node.js
  - **PHP servers**: PHP-based REST frameworks and server utilities
  - **Symfony2-specific** entries: bundles and components tailored for building REST services with the Symfony ecosystem

- **Collaboration and contribution**
  - Public repository with contribution guidelines (`contributing.md`)
  - Licensed under CC0-1.0, allowing broad reuse of the curated list
