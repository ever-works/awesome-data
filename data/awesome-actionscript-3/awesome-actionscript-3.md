# Awesome ActionScript 3

**URL:** https://github.com/robinrodricks/awesome-actionscript3#readme  
**Category:** Themed directories  
**Tags:** programming-languages, actionscript, awesome-lists

## Overview
Awesome ActionScript 3 is a curated, community-maintained directory of libraries, components, tools, and resources for the ActionScript 3 programming language and Adobe AIR runtime. It focuses on helping developers build cross‑platform desktop and mobile applications and games with AIR and AS3.

## Key Focus
- ActionScript 3 language ecosystem
- Adobe AIR desktop and mobile app development
- Game and graphics development (including GPU‑accelerated workflows)
- Extensions and native integrations via Air Native Extensions (ANEs)

## Features
- **Curated Resource List**  
  A structured collection of notable libraries, frameworks, components, and tools for ActionScript 3 and Adobe AIR.

- **Adobe AIR & AS3 Coverage**  
  Resources oriented around:
  - Cross‑platform desktop and mobile application development using Adobe AIR
  - Game development in ActionScript 3
  - Use of AIR’s built‑in features (file system, SQLite, sensors, etc.)

- **Support for Native Integrations (ANEs)**  
  Highlights and supports the pattern of extending AIR with ANEs written in native languages such as:
  - VC++ for Windows
  - Java for Android
  - Swift/Objective‑C for iOS

- **Graphics & UI Framework Ecosystem**  
  Points developers to popular frameworks and UI toolkits, such as:
  - **Starling** for GPU‑rendered 2D graphics and games
  - **Feathers UI** as an optional UI framework on top of Starling

- **Development Tools Section**  
  Includes both commercial and free/open source tools, organized into subcategories (placeholders in the repository that are populated via the curated list):
  - Code Editors
  - Live Debuggers
  - Asset Creators
  - SWF Obfuscators
  - SWF Inspectors

- **Community Contributions**  
  Open contribution model where users can request additions or updates via GitHub Issues and pull requests, guided by a CONTRIBUTING document.

- **Supplementary Docs**  
  Additional markdown files such as `EXTRAS.md` for extra links or categories and a clear `LICENSE` for reuse of the list.
