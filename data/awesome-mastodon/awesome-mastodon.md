# Awesome Mastodon

A curated "awesome list" of tools, clients, libraries, and resources for the Mastodon decentralized microblogging network.

**Website:** https://github.com/hyperupcall/awesome-mastodon#readme  
**Category:** Themed Directories  
**Tags:** awesome-lists, social, open-source

---

## Overview

Awesome Mastodon is an open-source, community-maintained directory that aggregates high-quality Mastodon-related projects and learning materials. It focuses on the broader Mastodon ecosystem, from software and APIs to educational resources and hosting.

---

## Features

- **Curated resource collection**  
  - Focuses on high-quality, notable Mastodon-related projects and references.

- **Structured by topic** (as sections in the list):  
  - **Official** – Links to official Mastodon resources (e.g., main site, docs, source).  
  - **Instances** – Directories or lists of Mastodon servers/instances.  
  - **Statistics** – Tools and sites that provide metrics and analytics about Mastodon usage and growth.  
  - **People** – Notable individuals or accounts related to Mastodon (e.g., contributors, maintainers, key community members).  
  - **Tools** – Utilities and ancillary tools that integrate with or enhance Mastodon.  
  - **Desktop Clients** – Desktop applications for interacting with Mastodon.  
  - **Mobile Clients** – Mobile apps for accessing Mastodon on phones and tablets.  
  - **Libraries** – Programming libraries and SDKs for integrating with Mastodon’s APIs in various languages.  
  - **Servers** – Software and resources related to running Mastodon or compatible server software.  
  - **Guides** – How‑to articles, documentation, tutorials, and best practices for using or administering Mastodon.  
  - **Hosting** – Resources and providers for hosting Mastodon instances.  
  - **Similar Projects** – Links to related or alternative federated / social projects similar to Mastodon.  
  - **Bots** – Bot projects and automation tools built for Mastodon.

- **Open-source repository**  
  - Hosted on GitHub, with standard project files such as `LICENSE`, `CONTRIBUTING.md`, and `CODE-OF-CONDUCT.md` for community governance.

- **Community contributions**  
  - Public contribution guidelines (`CONTRIBUTING.md`) enable anyone to propose additions or changes via pull requests.  
  - Uses an Awesome List format, making it consistent with other lists under the awesome.re ecosystem.

- **Code of conduct**  
  - Explicit `CODE-OF-CONDUCT.md` to define expected behavior for contributors and maintainers.

- **License**  
  - Distributed under an open-source license (details in the `LICENSE` file in the repository).
