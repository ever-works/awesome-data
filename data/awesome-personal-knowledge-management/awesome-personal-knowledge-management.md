## Overview

Personal Knowledge Management (PKM) is the practice of capturing, organizing, and retrieving information to support learning, creativity, and productivity through structured note-taking and knowledge systems.

## Features

### Note-Taking Applications

#### Local-First and Privacy-Focused
- **Obsidian**: Markdown-based knowledge base
  - Bidirectional linking
  - Graph view visualization
  - Plugin ecosystem (1000+ plugins)
  - Local storage
  - Sync via iCloud/Dropbox or Obsidian Sync
  - Vim mode, canvas view
  - Mobile apps (iOS, Android)

- **Logseq**: Local-first outliner
  - Block-based editing
  - Daily notes
  - Graph visualization
  - Open source
  - PDF annotation
  - Flashcards and spaced repetition
  - Plugin system

- **Foam**: VSCode-based PKM
  - Markdown files
  - Git-friendly
  - Open source
  - Graph visualization
  - Templates and snippets

- **Zettlr**: Academic writing tool
  - Markdown editor
  - Citation management
  - Zettelkasten method
  - Export to multiple formats
  - Dark mode and theming

#### Cloud-Based Solutions
- **Notion**: All-in-one workspace
  - Databases and relations
  - Templates and blocks
  - Collaboration features
  - Web clipper
  - API integration
  - AI features

- **Roam Research**: Networked thought tool
  - Block references
  - Daily notes
  - Graph database
  - Bidirectional links
  - Query system
  - Subscription-based

- **RemNote**: Flashcard-integrated notes
  - Spaced repetition built-in
  - Bidirectional linking
  - Document editor
  - PDF annotation
  - LaTeX support

- **Mem**: AI-powered notes
  - Smart search
  - Auto-tagging
  - Chat interface
  - Time-based organization

#### Hybrid Approaches
- **Capacities**: Object-based notes
  - Typed content blocks
  - Local and cloud
  - Graph view
  - Daily notes

- **Tana**: Flexible workspace
  - Super tags
  - Live queries
  - Supertags and fields
  - AI integration

### PKM Methodologies

#### Zettelkasten
- Atomic notes (one idea per note)
- Unique identifiers
- Link-based connections
- Literature notes and permanent notes
- Index notes
- Developed by Niklas Luhmann

#### PARA Method
- **P**rojects: Active work with deadlines
- **A**reas: Ongoing responsibilities
- **R**esources: Topics of interest
- **A**rchives: Inactive items
- Created by Tiago Forte

#### Building a Second Brain (BASB)
- CODE framework:
  - **C**apture: Save valuable information
  - **O**rganize: Sort into PARA
  - **D**istill: Extract key insights
  - **E**xpress: Create outputs
- Progressive summarization
- By Tiago Forte

#### Evergreen Notes
- Notes that grow over time
- Densely linked
- Concept-oriented
- Refined through iteration
- Advocated by Andy Matuschak
