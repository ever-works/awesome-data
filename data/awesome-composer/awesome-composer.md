# Awesome Composer

**URL:** https://github.com/jakoch/awesome-composer#readme  
**Category:** Themed Directories  
**Tags:** PHP, Build Tools, Package Manager

## Overview
Awesome Composer is a curated, community-maintained directory of resources focused on Composer—the PHP package manager—and its ecosystem. It aggregates tools, plugins, scripts, services, tutorials, and more related to Composer, Packagist, and Satis.

## Features

### Scope & Coverage
- Curated list dedicated to Composer and its ecosystem
- Focus areas include:
  - Composer core usage and configuration (e.g., `composer.json`, `composer.lock`)
  - Packagist (the main PHP package repository)
  - Satis (static Composer repository generator)
  - Composer plugins
  - Composer-related tools
  - Composer scripts and script patterns
  - Services integrating with or enhancing Composer
  - Learning resources: tutorials, blogs, videos, and slides

### Composer & Ecosystem
- References and examples around key Composer files and concepts:
  - `composer.json` configuration
  - `composer.lock` usage
  - Integration with PHP extensions (e.g., `phpext_curl`)
  - Repository and package configuration (e.g., `url`, `path` repositories)
  - CLI utilities like `composer repl`
- Links to related awesome lists, such as **awesome-php**, for broader PHP ecosystem coverage.

### Plugins
- Collection of Composer plugins that extend Composer functionality.
- Examples of topics and patterns referenced:
  - Plugin interaction with `composer.json`
  - Use of `.gitattributes` in Composer-managed projects
  - Handling vendor directory (`vendor`) setup
  - Performance-related plugins (e.g., opcode warmup: `warmup-opcode`, `vendor/preload.php`).

### Tools
- Directory of tools that work with or build on Composer.
- Includes items that utilize or generate configuration files such as:
  - `composer.yml`
  - `composer.json`
- May cover build tools, automation utilities, project scaffolding, or dependency-analysis tools related to Composer.

### Scripts
- Collection of patterns and examples for Composer scripts defined in `composer.json`.
- References to:
  - Script sections such as `scripts` and `scripts-dev`
  - CI integration examples (e.g., `.travis.yml` with Composer scripts)
- Helps discover reusable script snippets for development, testing, deployment, or project maintenance.

### Services
- Section dedicated to online services that integrate with or enhance Composer, Packagist, or Satis (e.g., hosting, mirrors, automation, or repository-related services).

### Tutorials, Blogs, Videos, Slides
- Learning-focused sections organizing educational content:
  - **Tutorials:** Guides on installing, configuring, and using Composer, Packagist, and Satis.
  - **Blogs:** Articles covering Composer best practices, dependency management strategies, and ecosystem news.
  - **Videos:** Screencasts and conference talks about Composer workflows and tooling.
  - **Slides:** Slide decks from meetups and conferences on Composer-related topics.

### Packagist
- Information and resources around Packagist, the main PHP package repository.
- Includes:
  - General introduction to Packagist.
  - Guidance on setting up and using Packagist.

#### Packagist Mirrors
- Details and references related to Packagist metadata mirrors:
  - Information about Packagist metadata mirrors
  - Link to official metadata mirror documentation: `https://packagist.org/mirrors`
- Section on **Setup a Packagist Mirror** to help users configure or use mirror endpoints.

### Composer Repositories
- Section dedicated to Composer repository configuration and management.
- Topics include:
  - **Registry Manager:** tools or methods for managing multiple Composer repositories or registries.
  - **Private repositories:** examples and resources for hosting and consuming private Composer repositories.
  - **Private Packagist-like setups:** guidance around self-hosted or private package registries (e.g., using Satis or similar tools).

### Contribution
- Public contribution model via GitHub.
- Documented contribution process in `.github/CONTRIBUTING.md` to standardize how new resources are proposed and added.
