## Overview

The Big List of Naughty Strings (BLNS) is an evolving list of strings which have a high probability of causing issues when used as user-input data, intended for use in automated and manual QA testing.

## Features

- **Comprehensive Test Cases**: Newline-delimited strings covering multiple categories of problematic input
- **Security Focused**: Includes server-side code injection attempts, format string vulnerabilities, and other attack vectors
- **Unicode Coverage**: Special characters, zero-width characters, and Unicode edge cases
- **Multiple Formats**: Available in plain text, base64, and JSON formats
- **Language Libraries**: Wrappers available for .NET, Node.js, PHP, Python, and other languages
- **Integrated into Security Tools**: Included in SecLists and other security testing frameworks

## Use Cases

- Automated QA testing
- Manual security testing
- Fuzzing applications
- Input validation testing
- SQL injection prevention testing
- XSS vulnerability detection

## Security Warning

Intended for testing software you own and manage. Some strings can indicate security vulnerabilities; using them with third-party software may be illegal.
