# Developer-First Products

A curated, community-maintained directory of **developer-first products**—tools where developers are the primary users and decision-makers.

> Source: [GitHub – agamm/awesome-developer-first](https://github.com/agamm/awesome-developer-first)

---

## Overview

Developer-First Products is an open-source "awesome-style" list focused on cataloging tools built primarily for developers. It emphasizes products where the developer experience is central, covering areas like AI coding assistants, general developer tools, SaaS platforms targeting engineers, and more.

Category: **Themed Directories**  
Tags: `developer-tools`, `business-tools`, `saas`

---

## Features

- **Curated directory of developer-first tools**  
  Focuses specifically on products where developers are the main users and buyers.

- **Awesome-list format**  
  Organized as a public GitHub repository, following the common "awesome" list conventions.

- **Structured by categories**  
  Includes sections such as:
  - **AI Coding** – AI IDEs and AI assistants (e.g., tools like aider, cline are showcased in badges)
  - Additional categories exist in the README (e.g., other tool types and domains), organized for easy discovery.

- **Open contribution model**  
  - `CONTRIBUTING.md` file defines how to propose additions or changes.
  - Community-driven updates through pull requests and issues.

- **Change history & latest additions**  
  - Full commit history available for transparency.
  - Direct link to the latest additions via the repository’s commits page.

- **Clear licensing**  
  - `LICENSE` file in the repo specifies terms for using and contributing to the list.

- **Curation standards**  
  - `OUTCAST.md` suggests there are criteria for including or excluding tools (e.g., tools that no longer fit the “developer-first” definition or quality bar).

- **Developer-focused discovery**  
  Helps developers quickly find tools and SaaS products tailored to their workflows (coding, infra, collaboration, etc.).
